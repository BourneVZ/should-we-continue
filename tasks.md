# Should We Continue MVP 原子化实施任务列表

> 版本：v0.1  
> 生成日期：2026-06-26  
> 依据：`AGENTS.md`、`spec.md`、`specs/`、`plan.md`

## 执行约定

- 所有任务均为未完成状态。每项只创建或修改标题中列出的一个主要文件；执行时不得顺带重构其他文件。
- `[P]` 表示在其“依赖”全部完成后，可与其他同样满足依赖且不修改同一文件的任务并行执行。
- `RED` 任务必须先提交一个会失败的 Vitest 用例；不得使用 `.skip`、宽泛 mock 或先写实现让测试直接通过。实现任务只能在其对应 `RED` 任务存在后执行。
- `GREEN` 任务只修改指定实现文件，以最小实现让关联测试通过；不得通过删弱断言、改变测试语义或补造医学/政策事实来使测试变绿。
- UI 任务使用 **Vitest + `react-dom/server` 静态渲染** 检查安全可见性与信息顺序；不引入 React Testing Library 或 Playwright。交互逻辑必须先由 `domain` 或 `app-reducer` 的 Vitest 覆盖。
- 所有夹具均使用最小化虚构数据。不得将真实回答、自由文本、地址、医院信息、API Key、`.env.local` 内容或模型原始输出写入测试、快照或任务产物。
- 任何涉及医学、心理、法律、政策、经济成本或正式量表的任务，只能引用已在 `specs/research.md` 中登记且已核验的来源；缺少来源时记录为不可用并走启发式/通用文案降级，不能自行补写事实。

- `依赖：—`：可在当前阶段起点开始。
- `依赖：T-xxx`：必须在指定任务验收通过后开始。
- `门：Gx`：必须满足相应规格闸门；门未关闭时不允许启动代码、依赖安装或部署。

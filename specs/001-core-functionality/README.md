# specs

本目录用于存放 `should-we-continue` 的专项规格。

 `spec.md` 定义 MVP 的产品边界、核心流程、技术方向和实现前置条件；本目录下的文档用于在实现前补齐可测试、可追溯的细节。

## 实现前必须补齐的文档

- `research.md`：专业知识来源、规则依据、本地政策与成本配置依据。
- `questionnaire.md`：具体问卷模块、题目、选项、跳题逻辑和敏感题处理。
- `scoring.md`：红旗规则、维度等级、内部权重和不可展示分数规则。
- `personas.md`：角色化类型、形象边界、类型与维度的映射。
- `report.md`：个人报告和伴侣共同版报告模板。

## 约束

- `questionnaire.md`、`scoring.md`、`personas.md`、`report.md` 必须引用 `research.md`。
- 涉及医学、心理、法律、政策、经济成本的内容必须能追溯到来源或本地配置。
- 大模型只能在这些规格定义的框架中生成表达，不得自由发明核心规则。
- 用户可见报告不得展示总分、排名或“继续/中止适合度”。

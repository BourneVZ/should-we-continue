## 3. 实施范围（供确认）

用户确认后可开始的范围：

1. 使用 React、Vite、TypeScript、Tailwind、lucide-react 与 Vitest 建立本地优先的静态 MVP；
2. 按 `questionnaire.md` 3.6 实现声明式题库、回答状态和保存失败门禁；
3. 按 `scoring.md` 15 实现本地红旗、九维、确定性、推荐、路径条件和模板报告；
4. 按 `personas.md` 9 实现 12 题启发式角色映射及 R3/R4 抑制；
5. 按 `report.md` 9 实现稳定模板、共享过滤和无模型降级；
6. 按 `region.md` 实现只读静态卡片与受控、用户主动的刷新边界。

明确不在本次授权范围：账号、数据库、云端历史、使用分析、第三方错误监控、医疗/法律/心理诊断、去留裁决、未确认正式量表、无来源地区机构/价格、未经单独审查的视觉资产、自动联网抓取或部署。

## 4. Phase 1 前的测试清单

实现应以失败测试开始，并至少将下列规格转换为 Vitest 用例：

- 题库：所有 `answerKey` 唯一，选项 code 合法，所有规则引用已登记键；
- 安全：逐条 `RF-*` 边界、最高等级覆盖、R3/R4 抑制角色与共享；
- 评分：九维权重、缺失 60% 门槛、无已有子女默认高支持、人生发展低支持行动；
- 确定性：覆盖率、`uncertain`、`deferred`、本地矛盾和模型矛盾仅降一级；
- 推荐：固定优先级、并列按 `moduleId`、R3/R4 无常规推荐；
- 路径：每条 `PC-*` 的 `confirmed/pending` 与用户阅读状态不回写事实；
- 角色：最少题数、置信度 69/70、平分、兼容矩阵、禁止字段、R3/R4 固定抑制；
- 报告与共享：仅预定义内容 ID、无原始自由文本/总分/推荐、默认空授权、无授权占位；
- 地区：白名单、7 天缓存、冲突并列、失败不覆盖、地区字段绝不进入评分。

# Result Page Spec

更新时间：2026-06-28

## 1. 页面目标

结果页不是报告书，而是类型主页。

用户看完后应感觉：

- 这型像我
- 这句话扎到我了
- 这张图能代表我
- 我想继续点开别的类型看看

## 2. 固定结构

### 2.1 Hero

- 主视觉
- 代号
- 中文名
- 一句 punchline

### 2.2 你为什么是这一型

- 4-6 个特征标签
- 不显示精确分数

### 2.3 典型反应

- 用怀孕发现当下和未来养育想象来写

### 2.4 翻车点

- 写这型最容易失手的地方

### 2.5 需要别人怎么配合

- 从关系语言写支持偏好
- 不输出行动清单

### 2.6 相近类型

- 推荐 2-4 个类型

### 2.7 轻量指纹

- 5 大模型
- 每模型 3 维
- 每维仅展示低 / 中 / 高

## 3. 第 25 型结果页

`NOIS` 结果页必须同时做到：

- 足够好笑
- 不像程序报错
- 不像医学诊断
- 解释为什么“标准类型都差一点”

结果页需要额外显示：

- 你最接近的 3 个标准类型
- 一句校准提示

## 4. 禁止内容

- 医学建议
- 心理建议
- 去留判断
- 风险等级
- 真实用户隐私解释

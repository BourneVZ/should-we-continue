# Type System Spec

更新时间：2026-06-28

## 1. 总体结构

- 用户可见结果：24 个标准类型 + 1 个 `NOIS`
- 内部组织：8 个家族 x 每组 3 型
- 家族层默认不在前台强教育

## 2. 8 个家族的意义

家族不是医学事实，而是内部建模层。

它们代表更大的主矛盾：

- 控盘家族：事情还没开始，脑内已经跑完三轮风控
- 缓刑家族：不是没想法，是谁都别逼我现在给答案
- 验收家族：说支持可以，但先把清单拿出来
- 边界家族：不是冷，是不想被整个人吞没
- 献祭家族：还没开始，已经在想自己要让掉多少
- 投射家族：未来画面感太强，容易先爱上想象
- 地基家族：先看承重，再谈理想
- 失序家族：变量一多，系统就开始抖

## 3. 命名格式

每个标准类型包含：

- 4-5 字符短代号
- 中文类型名
- 一句 punchline

例如：

- `CTRL / 控盘总监 / 先别感动，我先把变量列完`
- `DRAG / 缓刑选手 / 不是不决定，是谁都别催我现在决定`

以上是格式样例，不是最终类型表。

## 4. 风格边界

允许：

- 轻毒舌
- 自嘲
- 夸张
- 生活细节

禁止：

- 病理化标签
- 对创伤、自伤、暴力、流产等严肃处境开玩笑
- 直接暗示“你不适合当妈妈”

## 5. 第 25 型

- 代号：`NOIS`
- 中文名：噪点综合体

角色方向：

- 不是“你很怪”
- 而是“你的脑内像多个反应同时开麦”
- 它的笑点来自回声、噪点、混响和自相矛盾

## 6. 视觉原则

- 同一世界观
- 同一画风
- 强人物识别
- 不靠仅换色区分
- 同家族可共享视觉母体
- 子型通过姿态、道具、表情和场景差异分化

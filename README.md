# should-we-continue

`should-we-continue` 是一个本地优先的决策支持工具，用于帮助用户梳理与妊娠继续或终止相关的支持条件、风险提示、沟通材料和行动清单。它不提供医疗、法律或心理诊断，也不替代线下专业支持。

## 运行

```bash
npm install
npm run dev
```

## 测试

```bash
npm test
```

## 构建

```bash
npm run build
```

## Vercel 环境变量

服务端可使用以下环境变量：

- `LLM_API_KEY`
- `LLM_BASE_URL`
- `LLM_MODEL_ANALYSIS`
- `LLM_MODEL_REPORT`
- `LLM_TIMEOUT_MS`
- `LLM_MAX_OUTPUT_TOKENS`

不要使用 `VITE_` 前缀暴露任何密钥到客户端。

## 模板降级

当未配置 Key、模型调用失败、输出超时或 schema 校验失败时，系统回退到本地模板模式。模板模式仍保留红旗、九维、路径条件和共享边界。

## 数据本地性

- MVP 默认不建账号，不使用云端数据库。
- 用户回答默认保存在当前浏览器本地。
- 同一设备上的其他使用者可能看到这些内容。
- 支持导出个人报告和共同讨论内容。
- 支持分别清除用户数据、伴侣数据或全部本机数据。

## 已知限制

- 未成年人妊娠不属于当前 MVP 可自动筛查的红旗，属于已知覆盖缺口。
- 地区信息仅作为带来源的参考卡片，不参与评分或路径建议。
- 共同讨论页只展示已授权摘要，不复制原始自由文本、量表数据或私密回答。

## 开发边界

- 不提供医疗建议、药物建议、手术建议或法律结论。
- 不引入第三方分析 SDK 或错误监控服务。
- 不在客户端存储或展示 API Key、原始模型输入输出或敏感诊断面板内容。
